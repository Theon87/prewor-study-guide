Prework Study Guide

HTML

  • The head element contains information about the page.
  • The body element represents the visible content shown to the user.
  • The meta elements in the head element is used by the browser or by other developers.
  • The link element creates the connection to an external file, like a css file.
  • The title in the head element is visible to the user in the browser tab.
  • It is important to use semantic elements in the body for screen readers.
  • The body is divided into header, main, and footer elements.
  • The header is can contain navigation bars.
  • The main section can contain section and p elements.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS is organized as selector & a declaration block which contains a property and value.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout main: switches to main branch.
  • git pull origin main: pulls the main branch from GitHub to the local machine.
  • Remember to use "git status" to check what branch we are on before coding.
  • code .:opens VS Code from GitBash.
  • The Git flow steps include; add, commit, push to feature branch. Then, open a PR and merge. Then close the GitHub issue.
  • git add -A: add changed files in our working directory to the staging area of Git.
  • git commit -m "change description message": commit staged changes to repository on GitHub.
  • Run "git pull origin main" before opening a PR.
  • git push origin feature/add-html: this is an example of how to push changes to GitHub.
